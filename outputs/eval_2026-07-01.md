## Key Differentiators

- **Format prescription solves automated checks**: v1's 18/20 auto score comes entirely from format compliance learned through CLAUDE.md patches. v2's regression to 12/20 is caused by format drift when the model is given a less prescriptive prompt — this shows the CLAUDE.md format instructions need to be embedded more rigidly in run_agent.py's prompt template.
- **Truncation is a pipeline failure, not a model failure**: The brief content was correct up to line 237; Sources and Methodology sections simply never appeared. Increasing `max_tokens` from 8192 to 16384+ is the single highest-leverage fix.
- **YAML frontmatter is a prompt ambiguity bug**: The model generated valid YAML between `---` markers because the prompt (before the fix) said "begin with the YAML frontmatter separator." This caused 4 regex-based checks to fail simultaneously by making the header items invisible to the markdown patterns.
- **v2 LLM parity despite format failures**: v2 and v1 achieved identical weighted LLM scores (46.6/60) because the data quality, quantitative accuracy, and action specificity were equally strong. The 12-point gap is purely automated checks.
- **General agent ceiling is real**: General agent cannot reach above ~40/100 without tool access, state memory, or format instruction — LLM intelligence alone does not substitute for structured data and EDB-specific format constraints.
- **Data Integrity gap is solvable with one formatting rule**: The difference between v2 (3/5) and v1 (5/5) on Data Integrity is that v1's calculation block explicitly states `EIBOR source: estimated (Base Rate + 15bps)` as its first line; v2's does not. This is one CLAUDE.md line fix.
